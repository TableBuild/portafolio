Proyectos (propuestos)

Ideas en fase de planeación: cada proyecto tendrá documentación, arquitectura y roadmap antes de implementación.

Blog histórico de fútbol

Tipo: Web App / SPA · Complejidad: Media

Aplicación web para gestionar contenido histórico del fútbol con autenticación de usuarios, publicación de artículos y administración de información mediante una arquitectura cliente-servidor.

  • Estado: Planeación (requisitos + diseño)
  • Arquitectura: Frontend + API + BD
  • Enfoque: CRUD, autenticación, roles
Angular Node.js MySQL

API REST de gestión de inventario

Tipo: Backend / API · Complejidad: Media

API REST en Python para gestionar inventarios con operaciones CRUD, validaciones, estructura modular y persistencia en PostgreSQL siguiendo buenas prácticas de backend.

  • Estado: Planeación (modelo de datos + endpoints)
  • Arquitectura: API REST + BD
  • Enfoque: Diseño de endpoints, capa de servicio, pruebas
Python PostgreSQL REST

Flashcards con IA

Tipo: SPA + IA · Complejidad: Alta

Aplicación web interactiva para apoyar el aprendizaje mediante flashcards generadas y reforzadas con IA, integrando frontend moderno con backend basado en APIs y modelo de generación.

  • Estado: Planeación (flujo UX + prompts + endpoints)
  • Arquitectura: SPA + API + persistencia
  • Enfoque: Integración IA, trazabilidad y control de costo
Angular FastAPI SQLite OpenAI API

Finanzas personales

Tipo: SPA / Plataforma · Complejidad: Alta

Aplicación web para registrar ingresos y egresos, administrar gastos fijos, generar reportes quincenales/mensuales/anuales y visualizar información mediante gráficos.

  • Estado: Planeación (alcance + módulos + seguridad)
  • Arquitectura: SPA + API + Firebase
  • Enfoque: Reportes, permisos, visualización de datos
Angular Python Firebase Charts
